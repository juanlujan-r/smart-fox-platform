# Smart Fox - Sistema de Diseño para Inversores
Contexto: Tailwind CSS 4, Lucide Icons.

## 1. Estándar Visual
Objetivo: Interfaz sobria, corporativa y consistente para stakeholders e inversores.

Paleta base (tokens sugeridos):
- `fox-900`: #111827
- `fox-800`: #1F2937
- `fox-700`: #374151
- `slate-600`: #475569
- `slate-500`: #64748B
- `fox-orange`: #FF8C00
- `red-600`: #DC2626
- `surface`: #FFFFFF
- `surface-muted`: #F8FAFC
- `border`: #E5E7EB

Tipografía:
- Títulos: `font-black` o `font-bold`, tracking tight.
- Cuerpo: `font-medium` para lectura rápida.
- Números/metricas: `font-mono` cuando aplique.

Sombras:
- Card principal: `shadow-lg` + borde sutil `border border-gray-100`.
- Modal: `shadow-2xl` + fondo de overlay `bg-black/40`.

Botones de acción (consistencia global):
- Primary (Fox-Orange): `bg-[#FF8C00] hover:bg-orange-600 text-white`.
- Secondary (Slate-600): `bg-slate-600 hover:bg-slate-700 text-white`.
- Danger (Red-600): `bg-red-600 hover:bg-red-700 text-white`.
- Tamaño: `py-3 px-4 rounded-xl font-bold`.
- Estados: `disabled:opacity-50 disabled:cursor-not-allowed`.

## 2. Componentes Críticos
Botón Global de Tienda:
- Acceso persistente a `/pos`.
- Sidebar: ítem fijo con ícono `ShoppingCart` y etiqueta “Punto de Venta”.
- TopBar: botón pequeño con ícono + texto corto “POS”.
- Prioridad visual: variante Secondary para no competir con acciones críticas.

Layout de tarjetas personales:
- Modo “solo lectura” por defecto, estilo deshabilitado.
- Inputs en modo lectura:
  - `bg-gray-50 text-gray-500 border border-gray-100 cursor-not-allowed`.
- Única excepción editable: contacto de emergencia.
- Indicar sección editable con un badge `bg-orange-100 text-orange-700`.

Formulario “Añadir Empleado”:
- Debe ser un Modal.
- Validación obligatoria:
  - EPS
  - ARL
  - Tipo de documento
  - Documento
  - Fecha de contratación
  - Cargo/Rol
- Error visual:
  - Texto: `text-red-600 text-xs font-semibold`.
  - Input: `border-red-300 ring-1 ring-red-200`.
- CTA principal: “Guardar empleado” con variante Primary.
- CTA secundaria: “Cancelar” con variante Secondary.

Vista de Certificados:
- Botón “Descargar PDF” con estilo corporativo limpio.
- Variante recomendada: Secondary con ícono `Download`.
- Ubicación: esquina superior derecha de la tarjeta o header de la vista.
- Evitar estilos llamativos; priorizar legibilidad y seriedad.

## 3. Accesibilidad
- Contraste mínimo AA en textos críticos.
- Focus states visibles en botones e inputs:
  - `focus:ring-2 focus:ring-[#FF8C00] focus:outline-none`.

## 4. Consistencia de Iconografía
- Usar Lucide Icons.
- Tamaño estándar: `w-5 h-5`.
- Íconos secundarios en `text-gray-400`, íconos de acción en `text-white`.

## 5. Notas de Implementación
- Mantener estilos coherentes con el login y dashboard actual.
- Evitar mezclas de variantes en un mismo flujo (ej. no usar Primary y Danger en acciones no destructivas).
